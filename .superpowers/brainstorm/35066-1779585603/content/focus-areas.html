HomeScreen + Widgets — Code Review

I found 4 distinct areas for improvement. Pick what matters most (select all that apply).

A

Unimplemented config options

Many settings are defined in types and exposed in settings forms but the widget renderers ignore them. Affects: Today (date format, weather summary, pages created today), Weather (UV index, sunrise/sunset, 3-day forecast), Focus Timer (break timer, completion sound, daily goal), Focus Queue (pinned items), Recent Work (folder exclusions), AI Planner (auto-refresh, calendar filter).

B

HomeScreen.tsx is a 1,245-line monolith

All 8 widget renderers, all state (weather, timer, calendar, day plan, resize drag…), and all helper logic live in a single file. Each widget should be its own component with its own state. This would make the file dramatically easier to work in and reduce bugs from state leaking between widgets.

C

No drag-to-reposition widgets

Widgets can be resized from corners but cannot be dragged to a new grid position. Users have to remove and re-add a widget to change its location. Adding drag-to-move would make the home center feel much more like a real dashboard.

D

New widget ideas

The catalog currently has 8 widget types. Potential additions: Habits tracker, Daily notes / scratch pad, Pomodoro with break tracking, Agenda (list view of upcoming events), Goals / OKRs progress, or a Bookmark launcher.
